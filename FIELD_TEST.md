새로운 기능은 이 문서에서 발견된 문제를 바탕으로 기획합니다.

## 2026-06-24

### 테스트 환경

- 버전 : v1.2
- 사용 시간 : 25분
- 등록 상품 수 : 12 / 48

### 확인된 장점

- 창고 ↔ 매대 왕복 횟수가 감소했다.
- 부족 상품을 한 번에 모아서 진열할 수 있었다.
- 자동완성 기능으로 일부 상품은 빠르게 등록 가능했다.
- 수량 기능으로 부족한 개수를 기록할 수 있었다.

### 발견된 문제

#### 1. 창고 없음 항목이 계속 남아있음

창고에 재고가 없는 것을 확인한 후에도 체크리스트에 계속 표시되었다.

실제 작업에서는 이미 확인이 끝난 항목이라 시야를 방해하는 느낌이 있었다.

#### 2. 자동완성 효과는 있었지만 기대만큼 크지는 않음

입력 시간이 조금 줄어들긴 했지만 체감상 큰 차이는 아니었다.

입력 과정 자체가 여전히 존재하기 때문에 작업 시간이 획기적으로 감소하지는 않았다.

#### 3. 신규 상품 등록 시 카테고리 자동 설정 문제

자동완성 목록에 없는 상품을 등록하면 카테고리가 자동 설정되지 않았다.

직접 카테고리를 수정해야 하므로 추가 입력이 발생했다.

#### 4. 반복적으로 부족한 상품 관리가 어려움

창고 재고가 없어 보충하지 못한 상품을 다음 근무에서도 다시 직접 입력해야 했다.

예시

- 포스틱
- 새우깡
- 특정 라면

- 창고 없음 기능은 실제 업무에서 활용되었다.
- 체크리스트를 활용한 보충 방식은 계속 유효했다.



### 발견된 문제



#### 1. 다른 점포에서는 미해결 상품이 오히려 방해가 됨

이전 점포의 미해결 상품이 그대로 남아 실제 업무와 맞지 않았다.

결국 직접 삭제해야 했으며, '모두 삭제' 기능이 없어 하나씩 삭제해야 했다.

#### 2. 창고 없음 체크는 즉시 하지 않았다

예상

상품을 찾다가 없으면 바로 체크

실제

창고 전체를 확인한 뒤 못 찾은 상품을 한꺼번에 처리했다.

기능은 적절했지만 실제 사용 순서는 예상과 달랐다.

#### 3. 체크리스트가 길어질수록 원하는 상품을 찾기 어려움

약 40개의 상품이 등록되자 체크리스트에서 원하는 상품을 찾는 시간이 증가했다.

병목이 '상품 등록'에서 '체크리스트 탐색'으로 이동하였다.

#### 4. 사용자 상품 DB를 수정할 수 없음

잘못 입력한 상품명이 즉시 저장되어 이후 자동완성에도 계속 표시되었다.

예시

`포카칩 양파맛ㅁ`

↓

자동 저장

↓

자동완성에 계속 표시

## 2026-06-28



### 테스트 환경

- 버전 : v1.4
- 같은 점포에서 이틀 연속 테스트
- v1.5 개발 중



### 좋았던 점

- 데이터가 쌓이면서 자동완성이 점점 편리해지고 있다.
- 사용자 상품 DB가 누적될수록 상품 입력 부담이 줄어드는 것이 체감된다.



### 불편했던 점



#### 1. 재확인 물품 기능의 효용이 낮음

같은 점포에서 이틀 연속 사용해도 재확인 물품 기능의 효용이 낮았다.

실제 업무는 창고부터 확인하는 것이 아니라 매대를 먼저 확인하는 방식이다.

또한 재확인 물품이 항목 추가 바로 아래에 위치하여 체크리스트 확인에 방해가 되었다.

결국 재확인 물품을 모두 삭제하고 작업을 시작했다.

#### 2. 다른 점포 데이터가 다시 나타나는 현상

3회전 정도 진행했을 때 이전에 다른 점포에서 사용했던 데이터가 새로고침 후 다시 나타났다.

원인은 아직 확인되지 않았으며 추가 검증이 필요하다.

#### 3. 사용자 상품 관리 화면 진입 버튼 없음

v1.5에서 사용자 상품 관리 화면은 구현되었지만 메인 화면에서 이동할 수 있는 버튼이 없다.

실사용에서는 접근성이 떨어진다.

#### 다음 개선 후보

우선순위 높음

- 메인 화면에 상품 관리 버튼 추가
- 재확인 물품 기능 제거 또는 후순위화

우선순위 중간

- 점포별 데이터 분리
- 새로고침 시 다른 점포 데이터가 나타나는 원인 확인



### 테스트 결론

자동완성은 데이터가 쌓일수록 실제 업무 효율이 높아지고 있다.

반면 재확인 물품 기능은 현재 업무 방식과 맞지 않아 효용이 낮았다.

앞으로는 재확인 물품보다 자동완성과 사용자 상품 관리 기능을 중심으로 개선하는 것이 우선이라고 판단된다.

### 3.긴 상품명 UI

상품명이 길어질 경우 수량 표시가 다음 줄로 내려가 가독성이 떨어졌다.

조치

## 불편했던 점

1. 사용자 상품 관리 페이지의 이동성이 낮음

사용자 상품 관리 페이지에 들어간 뒤 메인 화면으로 돌아가는 방법이 직관적이지 않았다.

브라우저의 뒤로가기에 의존해야 하는 구조라 처음 사용할 때 잠시 당황했다.

메인 화면으로 이동하는 버튼이나 링크가 있으면 좋을 것 같다.

1. 자동완성 결과와 상품 관리 흐름이 분리되어 있음

자동완성 목록을 보다가 잘못 등록된 상품을 발견하는 경우가 있었다.

하지만 그 자리에서는 수정하거나 삭제할 수 없어 사용자 상품 관리 페이지까지 이동해야 했다.

자동완성에서 오류를 발견한 순간 바로 수정할 수 없다는 점이 작업 흐름과 조금 어긋났다.

1. 직전 체크리스트는 충분히 검증하지 못함

이번 테스트는 업데이트 직후라 이전 체크리스트 데이터가 충분하지 않았다.

따라서 직전 체크리스트 재사용 기능은 추가 필드 테스트가 필요하다.

## 테스트 결론

이번 테스트에서는 기능을 추가한 것보다 기능을 줄인 효과가 크게 체감되었다.

재확인 상품과 창고 없음 기능을 숨기면서 화면이 단순해졌고 실제 업무 흐름도 자연스러워졌다.

또한 수량 직접 입력은 많은 수량을 기록해야 하는 상황에서 확실한 효율 향상을 보여주었다.

반면 자동완성과 사용자 상품 관리 기능은 아직 서로 연결성이 부족해, 잘못 등록된 상품을 발견했을 때 작업 흐름이 끊기는 문제가 확인되었다.

다음 개선에서는 자동완성을 단순 입력 보조 기능이 아니라 사용자 상품 관리의 진입점으로도 고려할 필요가 있다.
