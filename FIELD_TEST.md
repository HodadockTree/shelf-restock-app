# 2026-06-24

### 테스트 환경

- 사용 시간: 25분
- 등록 상품 수: 12 / 48
- 버전: v1.2

---

### 좋았던 점

- 창고 ↔ 매대 왕복 횟수가 감소했다.
- 부족 상품을 한 번에 모아서 진열할 수 있었다.
- 자동완성 기능으로 일부 상품은 빠르게 등록 가능했다.
- 수량 기능으로 부족한 개수를 기록할 수 있었다.

---

### 불편했던 점

#### 1. 창고 없음 항목이 계속 남아있음

창고에 재고가 없는 것을 확인한 후에도 체크리스트에 계속 표시된다.

실제 작업 시에는 이미 확인이 끝난 항목이라 시야를 방해하는 느낌이 있었다.

검토 사항

- 창고 없음 전용 섹션 분리
- 창고 없음 처리 시 자동 완료
- 창고 없음 목록 별도 관리

---

#### 2. 자동완성 효과는 있었지만 기대만큼 크지는 않음

입력 시간이 조금 줄어들긴 했지만 체감상 큰 차이는 아니었다.

입력 과정 자체가 여전히 존재하기 때문에 작업 시간이 획기적으로 감소하지는 않았다.

---

#### 3. 신규 상품 등록 시 카테고리 자동 설정 문제

자동완성 목록에 없는 상품을 새로 등록할 경우

- 카테고리가 자동 설정되지 않음
- 직접 카테고리를 수정해야 함

추가 입력 작업이 발생함.

---



#### 4. 반복적으로 부족한 상품 관리가 안 됨

이전에 창고 재고가 없어서 보충하지 못한 상품이 다음 근무에도 그대로 부족한 경우가 있었다.

현재는 같은 상품을 다시 직접 등록해야 한다.

예시

- 포스틱
- 새우깡
- 특정 라면

등이 며칠 연속 부족한 경우 발생 가능

# 2026-06-27



## 테스트 환경

- 사용 시간: 약 30분
- 버전: v1.4
- 이전 근무지와 다른 점포에서 테스트

---



## 좋았던 점

- 자동 완성 기능이 편했고, 신규 데이터 등록도 자연스러웠다.
- 창고 없음 기능 자체는 실제 업무에서 활용되었다.
- 체크리스트를 활용한 보충 방식은 계속 유효했다.

---



## 불편했던 점



### 1. 다른 점포에서는 미해결 상품이 오히려 방해가 됨

이번 근무는 이전과 다른 점포였다.

이전 점포의 미해결 상품이 그대로 남아 있어 실제 업무와 맞지 않았다.

결국 직접 삭제해야 했으며,

'모두 삭제' 기능이 없어 하나씩 삭제해야 하는 점이 번거로웠다.

검토 사항

- 재확인 리스트 전체 삭제
- 점포별 데이터 분리(장기 검토)
- 새로운 근무 시작 시 재확인 리스트 초기화 여부 선택

---



### 2. 창고 없음 체크는 즉시 하지 않았다

예상

상품을 찾다가 없으면 바로 "창고 없음" 체크

실제

창고 전체를 확인한 뒤

못 찾은 상품들을 한꺼번에 "창고 없음" 처리하는 경우가 많았다.

즉, 기능은 맞았지만 실제 사용 순서는 예상과 달랐다.

검토 사항

- 창고 없음 일괄 처리
- 다중 선택 기능
- 선택 항목 일괄 "창고 없음" 처리

---



### 3. 체크리스트가 길어질수록 원하는 상품을 찾기 어려움

약 40개의 상품이 등록되자

상품을 찾은 후

체크리스트에서 해당 상품을 다시 찾는 시간이 증가했다.

새로운 병목이

'상품 등록'

이 아니라

'체크리스트 탐색'

으로 이동했다.

검토 사항

- 완료 항목 자동 숨김
- 완료 항목 하단 이동
- 정렬 기능
- 검색 기능
- 카테고리 접기

---



### 4. 사용자 상품 DB를 수정할 수 없음

잘못된 상품명을 입력했을 때 바로 사용자 상품 DB에 저장되었다.

이후 자동완성 목록에도 즉시 나타나 수정하거나 삭제할 수 없었다.

예시

입력

"포카칩 양파맛ㅁ"

↓

자동 저장

↓

다음부터 자동완성 목록에 계속 표시

잘못 등록된 데이터도 계속 누적될 가능성이 있다.

검토 사항

- 사용자 상품 관리 화면 추가
- 상품명 수정 기능
- 상품 삭제 기능
- 저장 전 중복 및 오타 검증
- 첫 등록 후 되돌리기(Undo) 기능
