# 편의점 보충진열 체크리스트

## 프로젝트 개요

편의점 보충진열 업무를 돕는 모바일 웹앱.

매대를 순회하며 부족한 상품을 기록한 뒤, 창고에서 필요한 상품을 한 번에 수집하여 매대↔창고 왕복 횟수를 줄이는 것이 목적이다.

이 프로젝트는 실제 편의점 근무 중 계속 사용하며, 필드 테스트 결과를 바탕으로 기능을 개선한다.

## 현재 제품 방향

v1.7의 핵심 방향은 체크리스트 탐색성과 사용자 상품 관리 접근성 개선이다.

v1.6까지의 개선으로 수량 직접 입력, 직전 체크리스트 재사용, UI 단순화가 적용되었다. 특히 재확인 상품과 창고 없음 UI를 숨기면서 화면 복잡도가 줄었고, 실제 업무 흐름도 더 자연스러워졌다.

하지만 필드 테스트 과정에서 체크리스트가 길어질수록 원하는 상품을 찾는 시간이 새로운 병목으로 확인되었다. 이미 완료 상품은 아래로 이동하고, 라면 / 과자 카테고리 구분도 적용되어 있었기 때문에 v1.7에서는 정렬 기능이 아니라 체크리스트 내 검색 기능을 추가하였다.

또한 자동완성 목록에서 잘못 등록된 사용자 상품을 발견했을 때, 기존에는 사용자 상품 관리 페이지로 이동한 뒤 다시 검색해야 했다. v1.7에서는 사용자 상품 자동완성 항목에 관리 버튼을 추가하고, 해당 상품명이 검색된 상태로 관리 페이지에 진입하도록 개선하였다.

PWA는 기능 확장이 아니라 실제 근무 중 앱 접근성을 높이기 위한 개선으로 적용하였다. 단, LocalStorage 기반 앱의 안정성을 고려하여 오프라인 캐싱과 서비스워커 전략은 이번 범위에서 제외하였다.

## 기능 판단 기준

기능 유지 여부는 실제 편의점 근무 흐름을 기준으로 판단한다.

우선순위는 다음과 같다.

1. 체크리스트 작성 시간 단축
2. 반복 입력 감소
3. 실제 업무 흐름 유지
4. 단순한 UI
5. 유지보수하기 쉬운 구조

기능을 추가할 때는 다음을 우선 검토한다.

- 실제 근무 중 조작 횟수를 줄이는가
- 입력 또는 탐색 시간을 줄이는가
- 기존 업무 흐름을 방해하지 않는가
- UI가 복잡해지지 않는가
- LocalStorage 기반 구조에서 안정적으로 유지 가능한가

## 후순위 또는 제거 검토 기능



### 재확인 상품 기능

실제 업무 흐름과 맞지 않아 UI에서 숨김 처리하였다.

현재는 다음 버전에서 완전 제거 여부를 검토한다.

### 창고 없음 기능

창고 없음 여부를 즉시 확정하기 어려운 상황이 많고, 실제 업무 흐름에서 자주 사용되지 않아 UI에서 숨김 처리하였다.

현재는 다음 버전에서 완전 제거 여부를 검토한다.

### 오프라인 캐싱

v1.7에서 기본 PWA는 적용했지만, 오프라인 캐싱은 제외하였다.

LocalStorage 기반 앱에서 캐싱 전략을 잘못 적용하면 업데이트 후 이전 화면이 남거나 데이터 동기화 혼란이 생길 수 있으므로, 필요성이 명확해질 때 별도 검토한다.

## 다음 검증 항목



### 체크리스트 검색

- 긴 체크리스트에서 원하는 상품을 더 빨리 찾을 수 있는지
- 검색 중 체크 / 삭제 / 수량 변경이 자연스럽게 작동하는지
- 검색 기능이 실제 근무 중 자주 사용되는지
- 검색창이 화면을 복잡하게 만들지 않는지



### 자동완성 관리 흐름

- 자동완성에서 잘못 등록된 사용자 상품을 발견했을 때 관리 버튼을 사용하는지
- `/user-products?keyword=상품명` 이동이 실제로 수정/삭제 시간을 줄이는지
- 기본 상품 DB와 사용자 상품 DB 구분이 자연스러운지
- 자동완성 목록에서 관리 버튼이 방해되지 않는지



### 사용자 상품 관리 UX

- 카드 내부 수정 방식이 기존 하단 수정 폼보다 직관적인지
- 수정 / 삭제 버튼 스타일이 모바일에서 사용하기 편한지
- 뒤로가기 화살표만 남긴 내비게이션이 충분히 이해되는지



### PWA

- 홈화면 설치가 실제 접근성을 높이는지
- 독립 실행 화면이 업무 도구처럼 느껴지는지
- 모바일에서 앱 실행 흐름이 브라우저 접속보다 빠른지



### 직전 체크리스트

- 같은 점포 연속 근무에서 직전 체크리스트 재사용 빈도
- 전체 불러오기와 선택 추가 중 어떤 방식이 더 편한지
- 직전 체크리스트 개별 삭제가 필요한 상황이 얼마나 자주 발생하는지
