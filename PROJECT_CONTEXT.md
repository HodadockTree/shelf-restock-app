# 편의점 보충진열 체크리스트

## 프로젝트 개요

편의점 보충진열 업무를 돕는 모바일 웹앱.

매대를 순회하며 부족한 상품을 기록한 뒤,

창고에서 필요한 상품을 한 번에 수집하여

매대↔창고 왕복 횟수를 줄이는 것이 목적.

### 실사용 환경

사용자는 편의점 2개 지점을 번갈아 근무한다.

현재는 점포 구분 기능 없이 사용 중이며,

점포별 재고 차이는 아직 검증되지 않았다.

점포 분리는 실제 사용 데이터를 확보한 후 검토한다.

## 현재 문제

현재 가장 큰 병목은 신규 상품 등록 과정.

현재

상품명 입력  
→ 자동완성 검색  
→ 없으면 직접 입력  
→ 추가

문제

직접 입력한 상품은 자동완성 DB에 반영되지 않는다.

결과적으로

자주 등록하는 상품도 반복 입력이 발생한다.

다음 목표

사용자 상품 DB 자동 구축

사용자가 입력한 상품을 자동 저장하여

자동완성 품질을 지속적으로 향상시키는 것.
