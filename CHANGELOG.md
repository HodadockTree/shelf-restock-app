# CHANGELOG

## v1.0 - 최초 MVP

### 목표

편의점 보충진열 시 창고↔매대 왕복 횟수 감소

### 구현

- 상품 추가
- 카테고리 분류
- 우선순위 선택
- 체크리스트 생성
- 완료 체크
- LocalStorage 저장

### 결과

실사용 가능한 최소 기능 구현 완료

### 발견한 문제

- 상품명을 직접 입력해야 함
- 입력 시간이 오래 걸림
- 보충 목록 작성 자체가 새로운 업무가 됨

## v1.1 - 입력 속도 개선

### 목표

상품 입력 시간 단축

### 구현

- 상품 자동완성 기능 추가
- 카테고리 데이터 정리
- 상품 데이터 구조 개선

### 결과

- 상품명 입력 속도 향상
- 반복 입력 부담 감소

### 실사용 후기

#### 장점

- 창고↔매대 왕복 횟수 감소
- 필요한 상품을 한 번에 수집 가능

#### 단점

- 체크리스트 작성 시간이 여전히 길다
- 카테고리 선택이 번거롭다
- 상품 식별이 어려운 경우가 있다

### 발견한 문제

예시

포카칩 양파맛 3개 필요

창고에는 1개만 존재

현재 시스템에서는 표현 불가능

또한

도리토스  
나쵸  
포카칩

등 유사 상품 구분이 어려움

# v1.4 - 사용자 상품 DB 구축



## 목표

실제 근무 중 발견한 신규 상품을 자동으로 저장하여  
자동완성 품질을 지속적으로 향상시킨다.

## 구현

- 사용자 상품 DB(LocalStorage) 추가
- 신규 상품 자동 등록
- 기본 상품 + 사용자 상품 자동완성 통합
- 기본 상품 중복 저장 방지
- 상품 관련 로직 리팩토링
  - 전체 상품 조회
  - 자동완성 검색
  - 사용자 상품 등록



## 예시

기존

포카칩 양파맛 입력

↓

체크리스트 추가

↓

다음 근무

↓

다시 직접 입력

변경

포카칩 양파맛 입력

↓

사용자 상품 DB 자동 저장

↓

다음 근무

↓

"포" 입력

↓

자동완성 표시

## 결과

기존

새로운 상품은 매번 직접 입력해야 했다.

↓

개선

한 번 입력한 신규 상품은 다음부터 자동완성에 포함된다.

자동완성 데이터가 실제 사용에 따라 지속적으로 성장한다.

이전 근무에서 부족했던 상품을 재확인 가능
