# CHANGELOG

## v1.0 - 최초 MVP

### 목표

편의점 보충진열 시 창고↔매대 왕복 횟수 감소

### 구현

- 상품 추가
- 카테고리 분류
- 우선순위 선택
- 체크리스트 생성
- 완료 체크
- LocalStorage 저장

### 결과

실사용 가능한 최소 기능 구현 완료

### 발견한 문제

- 상품명을 직접 입력해야 함
- 입력 시간이 오래 걸림
- 보충 목록 작성 자체가 새로운 업무가 됨

---

## v1.1 - 입력 속도 개선

### 목표

상품 입력 시간 단축

### 구현

- 상품 자동완성 기능 추가
- 카테고리 데이터 정리
- 상품 데이터 구조 개선

### 결과

- 상품명 입력 속도 향상
- 반복 입력 부담 감소

### 실사용 후기

장점

- 창고↔매대 왕복 횟수 감소
- 필요한 상품을 한 번에 모을 수 있음

단점

- 체크리스트 작성 시간이 여전히 길다
- 카테고리 선택이 번거롭다
- 상품 식별이 어려운 경우가 있다

### 발견한 문제

예시

포카칩 양파맛 3개 필요

창고에는 1개만 존재

현재 시스템에서는 표현 불가능

또한

도리토스  
나쵸  
포카칩

등 유사 상품 구분이 어려움

## v1.2 - 수량 및 창고 상태 관리

### 목표

실제 보충 업무 상황 반영

### 구현

- 필요 수량 기능 추가
- 수량 +/- 버튼 추가
- 창고 없음 체크 추가

예시

신라면 툼바 x2

☑ 창고 없음

### 결과

기존

상품 존재 여부만 기록

↓

개선

필요 수량과 창고 상태 기록 가능

### 실사용 검증 예정

확인할 항목

- 수량 기능이 실제로 도움이 되는가
- 창고 없음 체크 사용 빈도
- 입력 시간이 증가하는가 감소하는가
- 수량 입력 방식이 적절한가
