# CHANGELOG

이 프로젝트는 실제 편의점 근무에서 사용하며, 실사용 테스트 결과를 바탕으로 기능을 개선하는 방식으로 개발됩니다.

---

## v1.0 - 최초 MVP

### 목표

편의점 보충진열 시 창고↔매대 왕복 횟수를 줄일 수 있는 최소 기능(MVP)을 구현한다.

### 구현

- 상품 추가
- 카테고리 분류
- 우선순위 선택
- 체크리스트 생성
- 완료 체크
- LocalStorage 저장

### 결과

- 실사용 가능한 최소 기능 구현 완료
- 실제 편의점 근무 테스트를 시작할 수 있는 기반 마련

### 다음 버전 과제

- 상품명을 직접 입력해야 하는 문제
- 입력 시간이 오래 걸리는 문제
- 자동완성 기능 필요

## v1.5 - 사용자 상품 DB 관리

### 목표

사용자 상품 DB를 안전하게 수정·삭제할 수 있는 관리 기능을 구현하고, 자동완성 데이터의 품질을 유지한다.

---

### Step 1 - 데이터 구조 개선 ✅

구현

- UserProduct에 id 추가
- crypto.randomUUID() 적용
- 기존 LocalStorage 자동 마이그레이션
- 마이그레이션 후 자동 재저장

결과

- 사용자 상품마다 고유 ID 부여
- 기존 데이터와 완전 호환
- 수정 및 삭제 기능 구현 기반 마련

---

### Step 2 - 사용자 상품 관리 API ✅

구현

- getUserProducts()
- searchUserProducts()
- isDuplicateProduct()
- updateUserProduct()
- deleteUserProduct()

결과

- CRUD API 구축 완료
- UI와 데이터 처리 로직 분리
- Firebase 이전을 고려한 API 구조 마련

---

### Step 3 - 사용자 상품 관리 화면 ✅

구현

- 사용자 상품 관리 페이지
- 상품 관리 진입 버튼
- 사용자 상품 검색
- 사용자 상품 수정
- 사용자 상품 삭제
- 중복 상품명 검사
- 기본 상품 DB와 사용자 상품 DB 중복 방지
- 자동완성 중복 제거

결과

- 잘못 등록한 사용자 상품을 직접 관리 가능
- 자동완성 데이터 품질 유지
- React Key 중복 오류 해결

---

### Step 4 - 실사용 테스트 및 개선 ✅

실사용 결과

- 사용자 상품 관리 기능 정상 동작
- 자동완성 데이터는 누적될수록 효율 향상
- 재확인 기능의 실사용 효용이 낮음을 확인
- 직전 체크리스트 재사용 기능의 필요성 확인

버그 수정

- 체크리스트 LocalStorage 저장 누락 수정
- 체크리스트 복원 버그 해결
- state와 LocalStorage 동기화 개선

다음 버전(v1.6) 검토

- 직전 체크리스트 재사용
- 상품 관리 UX 개선
- 긴 상품명 UI 개선
- 수량 입력 방식 재검토
- 날짜별 체크리스트 관리
