# 편의점 업무 도우미

편의점 근무 중 반복적으로 발생하는 업무를 돕기 위한 모바일 웹앱입니다.

현재는 다음 기능을 제공합니다.

· 보충진열 체크리스트  
· 폐기 메모  
· 기본 PWA

실제 편의점 근무 중 사용하며, 필드 테스트 결과를 바탕으로 필요한 기능을 추가하고 불필요한 기능은 줄이는 방식으로 개발합니다.

---

## 주요 기능

### 1. 보충진열 체크리스트

매대를 순회하며 부족한 상품을 기록한 뒤, 창고에서 필요한 상품을 한 번에 수집하여 매대↔창고 왕복 횟수를 줄이는 기능입니다.

구현 기능

· 상품 자동완성  
· 사용자 상품 DB 자동 구축  
· 체크리스트 관리  
· 필요 수량 입력  
· 수량 직접 입력  
· 체크리스트 내 검색  
· 체크리스트 초성 검색  
· 체크리스트 카테고리 접기/펼치기  
· 카테고리별 항목 수 표시  
· 직전 체크리스트 저장 및 불러오기  
· 사용자 상품 관리  
· 사용자 상품 검색  
· 사용자 상품 수정  
· 사용자 상품 삭제  
· 자동완성 초성 검색  
· 자동완성에서 사용자 상품 관리 페이지로 이동  
· 공통 검색 유틸 기반 검색  
· LocalStorage 자동 저장

---

### 2. 폐기 메모

내일 폐기 예정 상품이나 다음 근무자가 확인해야 하는 상품을 사진과 메모로 남기는 기능입니다.

구현 기능

· 사진 촬영 또는 사진 선택  
· 사진 미리보기  
· 간단한 메모 입력  
· 사진만 저장 가능  
· 메모만 저장 가능  
· 사진과 메모 함께 저장 가능  
· 저장된 폐기 메모 목록 보기  
· 작성 시각 표시  
· 삭제  
· LocalStorage 저장

---

### 3. 기본 PWA 지원

· 홈화면 설치  
· 앱 아이콘  
· 독립 실행  
· manifest 설정

단, 현재는 LocalStorage 기반 앱의 안정성을 우선하여 오프라인 캐싱과 서비스워커 전략은 적용하지 않았습니다.

## 개발 실행

### 1. 패키지 설치

```bash
npm install

```

### 2. 개발 서버 실행

```bash
npm run dev

```

브라우저에서 아래 주소로 접속합니다.

```txt
http://localhost:3000

```

폐기 메모 페이지는 아래 경로에서 확인할 수 있습니다.

```txt
http://localhost:3000/disposal-memos

```

## 실사용 테스트 예정

### 보충진열

· 체크리스트 초성 검색이 긴 목록 탐색 시간을 줄이는지 확인  
· 자동완성 초성 검색이 상품 추가 속도를 높이는지 확인  
· 체크리스트 카테고리 접기/펼치기가 특정 카테고리 집중 보기에 도움이 되는지 확인  
· 자동완성에서 사용자 상품 관리 페이지로 이동하는 흐름이 실제 업무 중 자연스러운지 확인  
· 사용자 상품 카드 내부 수정 방식이 기존 방식보다 직관적인지 확인  
· PWA 설치 후 실제 근무 중 접근성이 개선되는지 확인  
· 직전 체크리스트 재사용 기능이 같은 점포 연속 근무에서 유용한지 확인

### 폐기 메모

· 사진 촬영 흐름이 실제 근무 중 빠른지 확인  
· 사진과 메모를 함께 저장하는 방식이 공책 수기 작성보다 편한지 확인  
· 사진만 저장하는 방식이 실제 업무에 유용한지 확인  
· 메모만 저장하는 방식이 실제 업무에 유용한지 확인  
· 작성 시각 표시만으로 충분한지 확인  
· 폐기 예정일이나 위치를 별도 필드로 입력할 필요가 있는지 확인  
· 저장된 메모가 쌓였을 때 목록 확인이 불편하지 않은지 확인  
· 삭제만으로 충분한지, 완료 처리 기능이 필요한지 확인  
· LocalStorage에 사진을 여러 장 저장했을 때 속도나 용량 문제가 없는지 확인

## 프로젝트 원칙

기능 추가보다 실제 업무 효율을 우선합니다.

우선순위는 다음과 같습니다.

1. 실제 업무 시간 단축
2. 반복 입력 감소
3. 실제 업무 흐름 유지
4. 단순한 UI
5. 유지보수하기 쉬운 구조
6. 안정적인 LocalStorage 사용

기능은 반드시 실제 편의점 근무에서 검증한 후 다음 버전을 설계합니다.

필요성이 확인되지 않은 기능은 먼저 추가하지 않고, MVP로 실사용한 뒤 FIELD TEST를 바탕으로 판단합니다.
